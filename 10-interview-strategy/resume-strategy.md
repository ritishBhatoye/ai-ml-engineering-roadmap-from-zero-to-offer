## The Format (Non-Negotiable)

- **1 page.** No exceptions for internship/entry-level.
- **No photo, no objectives statement, no hobbies.**
- **Clean layout.** Use a LaTeX template (Overleaf) or Jake's Resume template.
- **PDF only.** Never Word.

## Rules for Writing Bullet Points

### The Formula: Action + What + How + Impact

**Bad:** "Worked on an ML project for spam detection."

**Good:** "Built a spam classifier using TF-IDF + Logistic Regression on 50K emails, achieving 0.94 F1-score, deployed as a REST API with FastAPI and Docker."

### Quantify Everything

- "Improved F1-score from 0.72 to 0.84 using feature engineering"
- "Reduced inference latency by 40% through model quantization"
- "Processed 500K+ records using optimized Pandas pipelines"

### Power Verbs for ML Resumes

Built, Developed, Implemented, Engineered, Deployed, Optimized, Reduced, Improved, Designed, Automated, Analyzed, Evaluated

---

## What Screeners Look For (< 30 Seconds Per Resume)

1. **Projects section** — Do they exist? Are they real (with GitHub links)?
2. **Technical skills** — Does it match the JD?
3. **Quantified impact** — Numbers stand out immediately
4. **GitHub link** — No link = no projects verified
5. **Clean formatting** — Messy resume = messy engineer
