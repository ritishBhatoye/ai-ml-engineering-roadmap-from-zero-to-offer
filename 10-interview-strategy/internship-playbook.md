## The Two Tracks

| Dimension | Startups | Product Companies (FAANG-level) |
|---|---|---|
| **What they test** | Speed, breadth, can you ship? | Depth, rigor, system thinking |
| **Interview format** | Take-home project + discussion | Multiple rounds: coding, ML theory, system design |
| **Resume focus** | End-to-end projects, deployment | Strong fundamentals, research exposure |
| **Timeline** | Fast (1-2 weeks) | Slow (1-3 months) |
| **Bar for intern** | Can you build things independently? | Do you have strong foundations to grow? |

---

## Strategy for Startups

### What They Want
- Someone who can go from data → model → API in a week
- Comfort with ambiguity and messy data
- Deployment skills (Docker, APIs, basic cloud)

### How to Prepare
1. Build 1-2 end-to-end projects (data → model → deployed API)
2. Learn FastAPI + Docker well
3. Practice with messy, real-world datasets (not clean Kaggle competitions)
4. Be ready to discuss product thinking: "How would this model help the business?"

### How to Apply
- Apply directly on startup career pages (AngelList/Wellfound, LinkedIn)
- Cold email founders with a link to a relevant project on your GitHub
- Show you can start contributing immediately

## Strategy for Product Companies

### What They Want
- Strong ML fundamentals (they'll teach you the rest)
- Clean problem-solving under pressure
- Good communication and structured thinking
- Potential to grow into a senior engineer

### How to Prepare
1. Master ML theory deeply (all topics in `03-ml-fundamentals/`)
2. Practice 50+ ML coding problems
3. Learn ML system design framework (7-step approach)
4. Do 5+ mock interviews
5. Polish resume with quantified project impact

### How to Apply
- University career portal (on-campus recruiting)
- Referrals (LinkedIn, alumni network)
- Online applications (2-3 months before internship start)
- Competitive programming profiles help (but not required)

## The Application Timeline

```
6 months before:  Start this roadmap
4 months before:  Begin applying to early-recruiting companies
3 months before:  Apply broadly (50+ applications)
2 months before:  Interview sprint (mock interviews, coding practice)
1 month before:   Final rounds, offer negotiations
```

## After Getting the Interview

### Round 1: Coding (45–60 min)
- ML implementation from scratch OR general DSA with ML flavor
- Practice: `06-ml-coding/coding-questions.md`

### Round 2: ML Theory (45–60 min)
- Deep questions on fundamentals
- Practice: `10-interview-strategy/interview-questions.md`

### Round 3: System Design (30–45 min, sometimes combined)
- "Design an ML system for X"
- Practice: `08-system-design/ml-pipeline-design.md`

### Round 4: Behavioral (30 min)
- STAR format: Situation, Task, Action, Result
- Prepare 3 stories: a challenge, a failure, a leadership moment

---

## Numbers That Matter

- **Applications sent:** 30–50 minimum
- **Response rate:** 10–20% for cold applications
- **Referral response rate:** 40–60%
- **Interview-to-offer conversion:** 20–30%
- **Expected timeline:** 2–4 months from first application to offer

> **Key insight:** The application process is a numbers game. Don't put all hopes on one company. Apply broadly, learn from each interview, and iterate.
